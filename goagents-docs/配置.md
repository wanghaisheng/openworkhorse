# 🏗️ PicoClaw 的 Workflow、Phase、Team、Task 配置系统设计

基于现有 PicoClaw 架构和我们的变更计划，我来详细设计配置系统。

## 📁 配置文件结构

### 整体目录结构
```bash
~/.picoclaw/workspace/
├── .goagents/                    # Go Agents 配置根目录
│   ├── config.yaml               # 全局配置
│   ├── workflows/               # 工作流定义
│   ├── phases/                  # 阶段定义（包含里程碑）
│   ├── teams/                   # 团队配置
│   ├── tasks/                   # 任务模板
│   ├── milestones/              # 里程碑定义
│   └── registry/                # 注册表和索引
├── skills/                     # 技能实现
└── memory/                     # 记忆存储
```

## 🎯 完整层级结构

基于核心模型的完整层级结构：
```
Workflow
  -> Phase
    -> Milestone          # 新增：里程碑层级
      -> Task
        -> Team
          -> Team Role
            -> Team Member Agent
```

# 🎯 PicoClaw 配置系统设计总结

## 📋 完整的配置架构

我已经为 PicoClaw 设计了完整的 **Workflow、Phase、Team、Task 配置系统**，基于现有架构实现零侵入性集成。

## 🏗️ 核心设计特点

### 1. **零侵入性架构**
- 完全基于现有技能系统
- 配置文件独立于 PicoClaw 核心
- 通过技能加载机制动态集成

### 2. **分层配置结构**
```
.goagents/
├── config.yaml          # 全局配置
├── workflows/           # 工作流定义
├── phases/              # 阶段配置
├── teams/               # 团队配置
├── tasks/               # 任务模板
├── registry/            # 配置管理器
└── cli/                 # 命令行工具
```

### 3. **智能加载机制**
- **多路径搜索**: 支持多个配置搜索路径
- **缓存优化**: 5分钟 TTL，提升性能
- **热更新**: 配置变更自动清除缓存

## 🎯 与 PicoClaw 集成

### 技能系统集成
```go
// PO Core 技能中加载配置
func (po *POCore) loadWorkflow(workflowID string) error {
    config, err := po.configManager.LoadWorkflow(workflowID)
    if err != nil {
        return err
    }
    po.currentWorkflow = config
    return nil
}
```

### 运行时配置

- **热更新**: 支持配置热更新
