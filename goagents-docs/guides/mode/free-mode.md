# 🆓 Free 模式指南

## 🎯 概述

Free 模式是 Go Agents v2.0 的灵活执行模式，适用于探索性强、不确定性高的项目。该模式让 Phase Lead 能够主导动态规划和任务分解，通过配置驱动实现灵活的项目管理。

## 🏗️ 模式特点

### 核心特征
- **动态规划**: Phase Lead 主导的动态任务规划
- **配置驱动**: 基于配置文件的灵活执行机制
- **探索优先**: 支持探索和实验的工作方式
- **适应性**: 根据项目进展动态调整策略

### 适用场景
- **创新项目**: 需要探索和验证的创新项目
- **研发项目**: 技术研发和概念验证项目
- **原型开发**: 快速原型和概念验证
- **不确定性项目**: 需求不明确的高不确定性项目

## 📋 执行流程

### 1. 配置初始化
```bash
# 查看可用Phase配置
picoclaw goagents phase list

# 选择Free模式配置
picoclaw goagents phase use free-phase

# 配置Free模式参数
picoclaw goagents config set mode=free
```

### 2. 动态任务规划
```bash
# 启动动态规划
@go "开发创新产品概念"

# PO 自动选择 Free 模式：
# 📋 **PO任务分析结果**
# 
# **需求**: 开发创新产品概念
# **推荐模式**: Free（创新探索项目）
# **动态规划**:
# - Phase Lead 主导任务分解
# - 根据探索结果动态调整
# - 支持实验和验证
# - 快速迭代和反馈
```

### 3. 灵活执行
```bash
# 动态调整任务
@go --adjust-task "根据用户反馈调整产品方向"

# 实验验证
@go --experiment "验证新概念的技术可行性"

# 快速迭代
@go --iterate "基于测试结果快速迭代优化"
```

## 🎯 Phase Lead 角色

### 规划职责
- **任务分解**: 根据项目特点动态分解任务
- **优先级调整**: 根据进展调整任务优先级
- **资源分配**: 灵活分配团队资源
- **风险管理**: 识别和管理项目风险

### 决策权限
- **任务选择**: 选择最适合的任务类型
- **执行方式**: 决定任务的执行方式
- **质量标准**: 设定适合的质量标准
- **时间安排**: 灵活安排时间计划

## 📊 质量保证

### 灵活质量标准
- **探索阶段**: 降低质量要求，鼓励创新
- **验证阶段**: 提高质量要求，确保可行性
- **开发阶段**: 标准质量要求，确保可靠性
- **交付阶段**: 严格质量要求，确保产品质量

### 动态质量门禁
- **阶段性检查**: 每个阶段设置适合的质量门禁
- **风险控制**: 识别和控制质量风险
- **持续改进**: 基于反馈持续改进质量
